When coding, write no comments by default and delete stale comments.
Only include a comment when the code is doing something non-obvious, and in that case, keep it brief.
For comments, focus on *why* the code is written in this manner or why it does some particular thing, rather than how.
Do not restate facts in the comment that are inferable from the code itself.
Do not create logical file sections using block comments.
Do not add comments at the top of source files to 'summarize' the file's contents
